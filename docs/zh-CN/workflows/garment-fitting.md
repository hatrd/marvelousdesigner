## 📁 关于 PZIP 文件

### 什么是 PZIP 文件？

!!! info "PZIP 文件基础知识"

    **PZIP = Marvelous Designer 专用的服装文件**

    - 保存了服装的版型（纸样）
    - 也包含缝线和物理设置
    - 可以轻松使用他人制作的服装
    - 是 VRChat 服装制作的强力起点

    **其中包含的信息**:
    - ✅ **版型数据**: 服装纸样
    - ✅ **缝合信息**: 哪些部分需要缝在一起
    - ✅ **物理设置**: 布料的硬度、重量等
    - ✅ **布局**: 版型的排列方式

## 🚀 实操：导入 PZIP 文件

### 步骤 1: 文件准备

!!! example "步骤 1-1: 确认并整理文件"

    **开始操作前的准备**:
    1. 将 PZIP 文件放入 `VRChat衣装制作/Garments/` 文件夹
    2. 确认文件名（建议使用英数字）
    3. 确认文件没有损坏

    **推荐文件名示例**:
    - ✅ `dress_summer_v2.pzip`
    - ✅ `casual_outfit.pzip`
    - ❌ `可愛いドレス.pzip` （避免使用日文）

### 步骤 2: 在 Marvelous Designer 中导入

!!! example "步骤 2-1: 导入 PZIP 文件"

    **基本导入步骤**:
    1. 在 Marvelous Designer 中打开已导入模型的项目
    2. 选择 **「File」** → **「Import」** → **「Garment」**
    3. 选择 PZIP 文件并点击 **「打开」**
    4. 等待导入完成（约 30 秒到 1 分钟）

    **确认导入成功**:
    - 2D 版型窗口中显示出版型
    - Object Browser 中新增服装名称
    - 此时先不要运行模拟

!!! tip "有多个 PZIP 文件时"
    一开始先用一个文件练习。等熟悉后再尝试同时导入多个文件会更稳妥。

### 步骤 3: 第一次运行模拟

!!! example "步骤 3-1: 先直接运行一次模拟"

    **先确认当前状态**:
    1. 按 **空格键** 开始模拟
    2. 观察服装穿到模型上的过程
    3. 确认出现了哪些问题
    4. 按 **空格键** 停止模拟

    **常见的初始状态**:
    - 😊 **运气好的情况**: 基本可以完美贴合
    - 😐 **常见情况**: 尺寸不合适或有悬空
    - 😅 **需要调整的情况**: 尺寸差异很大或出现重叠

!!! warning "一开始不完美也没关系"
    从一开始就完全贴合并不常见，接下来会逐步调整。

## 🔧 实际进行适配调整

### 步骤 4: 调整尺寸

!!! example "步骤 4-1: 整体尺寸调整"

    **当与模型的尺寸差距较大时的处理方法**:

    **方法 1: 调整整个版型的缩放**
    1. 按 **「Ctrl + A」** 选中全部版型
    2. 选择 **「Edit」** → **「Transform」** → **「Scale」**
    3. 调整数值（例如：`0.9` 为缩小 10%，`1.1` 为放大 10%）
    4. 点击 **「Apply」**

    **方法 2: 调整单个版型**
    1. 单独选中需要调整的版型
    2. 在 2D 版型窗口中拖动版型的角
    3. 进行微调

    **调整参考**:
    - 理想状态是服装与模型身体之间保持约 1 到 3cm 的距离
    - 太紧会穿模，太松会不自然地悬空

!!! example "步骤 4-2: 局部调整"

    **调整特定部位（胸部、腰部、下摆等）**:

    **胸部贴合调整**:
    1. 选中与胸部相关的版型
    2. 用 **「Edit Pattern」** 工具微调形状
    3. 注意不要修改缝线（红线）

    **调整袖长**:
    1. 选中袖子的版型
    2. 选中下摆部分后拖动调整
    3. 左右袖子按相同方式调整

### 步骤 5: 调整位置

!!! example "步骤 5-1: 设置版型初始位置"

    **布置到模型周围**:
    1. 选择 **「3D Garment」** → **「Reset 2D Arrangement」**
    2. 版型会自动排列到模型周围
    3. 必要时再手动微调位置

    **手动摆放的要点**:
    - 躯干部分的版型放在模型的正面和背面
    - 袖子放在左右手臂附近
    - 裙摆部分放在腰部周围

### 步骤 6: 缝合与最终调整

!!! example "步骤 6-1: 执行自动缝合"

    **确认并执行缝线**:
    1. 确认红线（Internal Line）就是缝线
    2. 点击 **「2D Pattern」** → **「Free Sew」**
    3. 系统会自动执行缝合
    4. 如果出现报错，检查对应位置

!!! example "步骤 6-2: 最终模拟"

    **确认完成状态**:
    1. 按 **空格键** 开始模拟
    2. 确认模型动作时服装的运动表现
    3. 如果有不自然的地方，暂停后进行调整
    4. 重复调整直到结果满意

## 🎨 用于质量提升的调整

### 物理设置优化

!!! example "调整布料特性"

    **适合 VRChat 的设置**:
    1. 选中版型并查看 Property Editor
    2. 调整 **「Physical Property」**：
       - **Fabric Preset**: Cotton（棉）比较稳妥
       - **Particle Distance**: 5-10（兼顾质量与处理速度）
       - **Bend**: 0.1-0.5（布料硬度）

    **调整原则**:
    - 在 VRChat 中轻量化很重要
    - 避免过于真实的物理效果
    - 优先保证稳定运行

### 外观调整

!!! example "设置颜色与纹理"

    **基础外观调整**:
    1. 选中版型并查看 Property Editor
    2. 用 **「Color」** 设置基础颜色
    3. 用 **「Opacity」** 调整透明度（按需）

    **应用纹理**（进阶）:
    - 后续阶段再在 Unity 中设置更详细的纹理
    - 在 Marvelous Designer 阶段只设置基础颜色就足够

## 📤 导出用于 VRChat

### 步骤 7: 导出为 FBX 格式

!!! example "步骤 7-1: 导出设置"

    **适合 VRChat 的最佳导出方式**:
    1. 在满意的贴合状态下停止模拟
    2. 选择 **「File」** → **「Export」** → **「FBX」**
    3. 导出设置：
       - **Scale**: 1.0
       - **Format**: FBX 2020
       - **Include Avatar**: 取消勾选（模型另外处理）

    **保存位置**:
    - 放在 `VRChat衣装制作/Exports/` 文件夹中
    - 设置一个清晰易懂的文件名

## 🔧 故障排查

### 常见问题与解决方法

??? question "「无法打开 PZIP 文件」"
    **处理方法**:

    1. **确认文件格式**:
       - 确认扩展名是 `.pzip` 或 `.zip`
       - 确认文件没有损坏

    2. **重新确认导入方法**:
       - 按顺序选择 File → Import → Garment
       - 不是直接使用 File → Open

??? question "「看不到服装」"
    **处理方法**:

    1. **检查 Object Browser**:
       - 确认 Garments 区块中是否有服装名称
       - 通过 👁️ 图标切换显示/隐藏

    2. **运行模拟**:
       - 按空格键开始模拟
       - 可能是版型还没有转为 3D

??? question "「服装穿过了模型」"
    **处理方法**:

    1. **调整尺寸**:
       - 选中版型并放大
       - 用 Transform → Scale 进行整体调整

    2. **Collision 设置**:
       - Avatar → Edit Avatar → Collision
       - 调整模型的碰撞判定

??? question "「贴合感不好」"
    **处理方法**:

    1. **局部调整**:
       - 单独调整有问题部位的版型
       - 用 Edit Pattern 工具修正版型形状

    2. **分阶段调整**:
       - 不要试图一次就全部调到完美
       - 按“大致调整 → 细节调整”的顺序推进
