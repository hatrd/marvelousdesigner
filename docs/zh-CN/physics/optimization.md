## 🎯 理解 VRChat 的性能标准

### Performance Rating 系统

!!! info "VRChat Performance Rating 标准"

    **PC 版标准**：
    ```
    Excellent (绿): 所有项目都在推荐范围内
    Good (黄绿色): 基本都在推荐范围内，通常没有问题
    Medium (黄): 略重，部分用户可能会将其隐藏
    Poor (橙): 较重，很多用户会将其隐藏
    Very Poor (红): 非常重，几乎所有用户都会将其隐藏
    ```

    **Quest 版标准**（更严格）：
    ```
    Triangle Count: 7,500 以下 (PC: 20,000 以下)
    Material Slots: 1 个 (PC: 8 个以下)
    Texture Memory: 10MB 以下 (PC: 40MB 以下)
    ```

### 优化优先级

!!! example "高效优化的顺序"

    **Phase 1: 基础结构优化**（最重要）
    1. 减少并整合多边形数量
    2. 将材质数量压到最少
    3. 优化纹理尺寸

    **Phase 2: Marvelous Designer 优化**
    4. 调整 Particle Distance
    5. 将 Self Collision 降到最低
    6. 调整 Simulation Quality

    **Phase 3: Unity 集成优化**
    7. 合并并优化 Mesh
    8. 设置 LOD（进阶）
    9. 优化 Shader

## 📐 多边形优化策略

### 在 Marvelous Designer 阶段进行优化

!!! example "在 MD 阶段减少多边形"

    **通过 Particle Distance 控制**：
    ```
    高密度 (6-8): 20,000-40,000 triangles
    标准 (8-10): 10,000-20,000 triangles (推荐)
    低密度 (10-12): 5,000-10,000 triangles
    最轻量 (12-15): 3,000-5,000 triangles
    ```

    **高效设置 Particle Distance 的方式**：
    1. **复杂服装**：整体设为 8，仅细节部分设为 6，局部提高密度
    2. **简单服装**：整体统一设为 10
    3. **背景用服装**：设为 12-15，极限轻量化
    4. **重点服装**：以 8 为基础，必要时局部设为 6

### 在版型设计阶段进行优化

!!! example "高效的版型设计"

    **判断哪些部件可以合并**：
    ```
    推荐合并:
    - 前片 + 后片 (接缝较少时)
    - 袖子 (左右形状相同时)
    - 装饰部件 (小口袋、蝴蝶结等)

    保持分离:
    - 动态不同的部分 (上身 vs 裙摆)
    - 材质设置不同的部分
    - 颜色或纹理差异很大的部分
    ```

    **移除不必要的细节**：
    - 极小的装饰部件（小于 1cm）
    - 看不见的内部结构
    - 过于复杂的缝合线
    - 没有功能意义的装饰性接缝

## 🎨 材质与纹理优化

### 材质合并策略

!!! example "高效的材质管理"

    **合并的基本方针**：
    ```
    理想构成:
    - 主服装: 1-2 个材质
    - 复杂服装: 最多 3-4 个材质
    - 特殊服装: 只有在无法避免时才用到 5+

    合并方法:
    1. 颜色相近的部件共用同一材质
    2. 利用 UV 映射进行配色分区
    3. 在同一张纹理中划分区域
    ```

    **在 Unity 阶段合并材质**：
    1. **Material 合并**：合并使用相同 Shader 与设置的材质
    2. **创建 Texture Atlas**：把多张纹理合并成一张
    3. **删除无用 Material**：移除未被使用的材质

### 纹理优化

!!! example "优化纹理尺寸"

    **推荐纹理尺寸**：
    ```
    主纹理: 1024x1024 (标准)
    细节纹理: 512x512
    小物件与装饰: 256x256
    背景或不显眼部分: 128x128
    ```

    **压缩设置（Unity）**：
    ```
    PC 版: DXT1/DXT5 (高质量)
    Quest 适配: ASTC 6x6 (平衡)
    优先轻量化: ASTC 8x8 (最轻量)
    ```

    **纹理优化检查清单**：
    - [ ] 已缩小到必要的最小尺寸
    - [ ] 已选择合适的压缩格式
    - [ ] 已启用 Mip Maps 生成
    - [ ] 已删除不必要的 Alpha 通道

## ⚙️ 物理模拟优化

### 优先轻量化的物理设置

!!! example "面向 VRChat 优化的物理设置模板"

    **优先轻量化的设置**：
    ```
    Particle Distance: 10-12
    Self Collision: 完全禁用
    Collision Thickness: 最小值
    Simulation Quality: Low-Medium
    Iterations: 最小值
    ```

    **兼顾质量的轻量化设置**：
    ```
    Particle Distance: 8-10
    Self Collision: 仅在重要区域启用
    Collision Thickness: 1-2mm
    Simulation Quality: Medium
    Bend/Stretch: 保持标准值
    ```

### 局部优化技巧

!!! example "按部件划分的优化策略"

    **对看不见、不会动的部分极限轻量化**：
    ```
    鞋子内侧: Particle Distance 15+
    配饰: Self Collision 完全禁用
    内搭（叠穿下层）: 最小设置
    ```

    **保留重要部位的质量**：
    ```
    裙摆: 保持标准设置
    袖口: 使用适中的质量设置
    胸部贴合区域: 保证必要精度
    ```

## 🛠️ 在 Unity 集成阶段进行优化

### Mesh 优化

!!! example "在 Unity 中优化 Mesh"

    **Mesh 合并**：
    1. **静态合并**：整合不会动的部件
       - 使用 `GameObject.CombineMeshes()`
       - 可与材质合并同时使用
    2. **动态合并**：保留骨骼权重并进行整合
       - 使用 Skinned Mesh 合并工具

    **Mesh 压缩设置**：
    ```
    Mesh Compression: Medium (推荐)
    Optimize Mesh: 启用
    Read/Write: 禁用（轻量化）
    ```

### LOD 设置（高级技巧）

!!! example "Level of Detail 设置"

    **LOD 构成示例**：
    ```
    LOD 0 (近距离): 全质量
    LOD 1 (中距离): 多边形减少 50%
    LOD 2 (远距离): 多边形减少 75%
    LOD 3 (最远距离): 最简化
    ```

    **实现步骤**：
    1. 在 Unity 中添加 **LOD Group** 组件
    2. 准备按距离区分的网格
    3. 设置切换距离
    4. 在 VRChat 中测试并调整

## 📊 性能测量与分析

### 测量工具与方法

!!! example "性能测量实战"

    **在 Unity 中测量**：
    1. **Profiler Window**：CPU/GPU/Memory 使用量
    2. **Frame Debugger**：详细分析渲染过程
    3. **Statistics**：实时渲染统计

    **使用 VRChat SDK 测量**：
    1. **Performance Rating**：基础评价
    2. **Build & Test**：确认真实环境中的表现
    3. **VRChat Analytics**：实际使用数据

### 确认优化效果

!!! example "优化前后的对比方法"

    **测量项目**：
    ```
    Triangle Count: 记录修改前后的数值
    Material Slots: 材质数量变化
    Texture Memory: 纹理占用量
    Performance Rating: 评级变化
    FPS Impact: Unity Scene View 中的 FPS 变化
    ```

    **记录模板**：
    ```
    优化对象: [服装名]
    实施日期: 2025-08-08

    【修改前】
    Triangle Count: 25,000
    Materials: 6 个
    Performance: Medium

    【修改后】
    Triangle Count: 12,000 (-52%)
    Materials: 3 个 (-50%)
    Performance: Good

    【实施内容】
    - Particle Distance: 6 → 8
    - 材质合并: 6 → 3
    - 纹理压缩: 2048 → 1024
    ```

## 🎯 实用优化工作流

### 分阶段优化步骤

!!! example "高效的优化流程"

    **步骤 1：分析现状**（15 分钟）
    1. 确认 VRChat SDK Performance Rating
    2. 找出主要问题项目
    3. 设定优化目标

    **步骤 2：确定优先级**（10 分钟）
    ```
    高优先级: Triangle Count, Material Slots
    中优先级: Texture Memory, Shader Quality
    低优先级: 细部微调项目
    ```

    **步骤 3：执行并验证**（60 分钟）
    1. 从高优先级项目开始依次优化
    2. 每次修改后都通过 Build & Test 进行确认
    3. 如果出现问题，就回退到前一个阶段

    **步骤 4：最终微调**（15 分钟）
    1. 检查整体平衡
    2. 在 VRChat 中测试实际表现
    3. 记录并保存设置内容

### 平衡质量与轻量化

!!! example "平衡调整的判断标准"

    **可以妥协的质量项目**：
    - 看不见部位的细节度
    - 极小装饰部件的保留
    - 过度高精度的纹理
    - 使用频率很低的特殊效果

    **必须保留的质量项目**：
    - 主要区域的轮廓
    - 靠近面部区域的质量
    - 动画播放时的自然感
    - 基本物理行为

## 🔧 故障排查

### 优化相关问题的解决方法

??? question "“优化后外观变差了”"
    **处理方法**：

    1. **分步回退**：逐项恢复设置，找出原因
    2. **局部优化**：不要整体一起动，只优化没有问题的部分
    3. **改用其他方法**：尝试不同的优化思路
    4. **重新审视质量标准**：确认真正需要保留的质量等级

??? question "“Performance Rating 没有改善”"
    **处理方法**：

    1. **逐项确认**：详细检查到底是哪项超过标准
    2. **排查隐藏问题**：如纹理内存、Shader 复杂度等
    3. **检查其他因素**：头像本体或其他组件的影响
    4. **分阶段处理**：执行更大胆的优化

??? question "“轻量化过头，物理表现变怪了”"
    **处理方法**：

    1. **调整 Particle Distance**：从轻量化设置稍微往质量侧回调
    2. **局部提升质量**：只恢复重要区域的设置
    3. **重新微调物理参数**：对 Bend/Stretch 做细微调整
    4. **重新评估 Simulation Quality**：必要时改回 Medium

### 从根本上解决性能问题

!!! example "根本性问题的处理思路"

    **从设计阶段开始优化**：
    - 一开始就按 VRChat 标准来设计
    - 避免做出过于复杂的结构
    - 用最低必要的细节级别进行制作

    **改进工具与工作流**：
    - 利用自动优化脚本
    - 系统化管理预设设置
    - 使用检查清单进行确认
