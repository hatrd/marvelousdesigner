# 中文翻译规则

## 目标

为 `docs/` 下日文教程生成简体中文版本，并保证同一概念在全站只有一种主译法。

## 术语规则

1. 译文必须优先命中 `translation/zh-CN/glossary/*.yml` 的固定译法。
2. 如果命中多个相近译法，使用词条中 `preferred: true` 的项。
3. 若无官方中文译名：
   - 品牌、产品、SDK、组件名保留官方英文；
   - 普通概念翻为自然中文；
   - 首次出现可写成 `英文名（中文解释）`。
4. 一个日文源术语只能有一个主译法；别名只允许记录在 glossary 的 `aliases` 中。

## 文体规则

1. 面向中文读者，语气自然、直接，不保留日文敬语痕迹。
2. 不要把技术名词全部硬译；优先使用中文技术社区已稳定接受的说法。
3. 步骤、提示、警告、按钮文案保持可操作性。

## Markdown 规则

1. 保留原有标题层级、提示块、表格、列表、代码块和相对链接。
2. 可以翻译链接文字，但不要修改目标路径。
3. 行内代码、命令、文件扩展名、类名、组件名保持原样。
4. HTML 片段保留标签结构，只翻译可见文本。

## 文件规则

1. 原文路径 `docs/foo/bar.md` 对应译文路径 `docs/zh-CN/foo/bar.md`。
2. 不要覆盖日文原文。
3. 新增固定译法时，先补 glossary，再改章节。
